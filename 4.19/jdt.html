Java Feature tab A new tab has been added that gathers cleanups that introduce the use of features from different Java versions.

Use those cleanups if you just have upgraded your JVM.

Multi-catch A new clean up has been added that refactors catch clauses with the same body to use Java 7's multi-catch.

The feature is enabled only with Java 7 or higher.

To apply the clean up, invoke Source > Clean Up..., use a custom profile, and on the Configure... dialog, select Use Multi-catch check box on the Java Feature tab.

Pull down common code from if/else statement A new clean up has been added that factorizes common code from all if / else if / else statements at the end of each blocks.

Ultimately it removes the empty and passive if conditions.

The statement matching does a deep analyze.

All the blocks may end with the same statements, or the blocks with different code may end with a jump statement (return, throw, continue or break).

To apply the clean up, invoke Source > Clean Up..., use a custom profile, and on the Configure... dialog, select Extract common code from if/else statement check box on the Duplicate code tab.

'Restricted identifiers' preference to color restricted identifiers A new preference 'Restricted identifiers' has been added under Java > Editor > Syntax Coloring , under the Java Category.

Some identifiers (for eg. var ,yield, record etc..) are restricted identifiers because they are not allowed in some contexts. Semantic highlighting options for such identifiers can be controlled by the language element Restricted identifiers under Java category. in the preference page Java > Editor > Syntax Coloring.
