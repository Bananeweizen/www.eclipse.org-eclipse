Views, Dialogs and Toolbar

Projects layout made hierarchical by default in Project Explorer To better handle multi-module, nested and hierarchical projects, the default Project Layout in Project Explorer view has been changed from Flat to Hierarchical.

You can still restore the layout to Flat in the usual ⋮ view menu.

Debug

Interpretation of form feed in Console View The interpretation of ASCII control characters in Console was extended to recognize the characters form feed (\f) and vertical tab (\v).

This feature is disabled by default. You can enable it on the Run/Debug > Console preference page.

Termination time in Console View The Console View label will now show the termination time of a process additional to the launch time.

Preferences

Preference for resource rename mode: inline or dialog This preference allows to configure resource renaming behavior in Project Explorer: either an inlined text field or a dialog. Without specifiying the preference, renaming is done inlined. The preference is found directly in the General preferences page.

The preference can also be specified via product customization:
  • org.eclipse.ui.workbench/RESOURCE_RENAME_MODE=inline
  • org.eclipse.ui.workbench/RESOURCE_RENAME_MODE=dialog

Note that renames that would affect more than 1 resource are always performed with a dialog.

Themes and Styling

More styling options for CTabItem On a CTabItem you can now set an individual foreground color and an individual selection foreground color.
Welcome screen in dark theme The dark theme setting is now taken into account by the welcome screen on macOS and Linux.

General Updates

Allow configuration of enabled SWT addons in application model Loading the addons for SWT automatically enabled all addons (Drag'n'Drop, MinMax, Split, Cleanup). Using tags (DisableDnDAddon, DisableMinMaxAddon, ...) on the application model, developers can now disable unwanted addons.
Redrawn turned off by default during collapse and expand operations in tree viewer To improve interactive performance, redraw is turned off by default during the collapse and expand operation of tree viewers. This significantly improves these operations compared to drawing the updates synchronously.
Interactive performance Interactive performance has been further improved in this release.
Hyperlink support in cheatsheet item description Now you can put hyperlinks in cheat sheet item description.

Hyperlinks are expressed using <a href="URL">diplay link</a>, where URL is a valid URL to an external web site or a CheatSheet URL that represents a CheatSheat action.

All cheat sheet URLs have the following form: http://org.eclipse.ui.cheatsheet/<action_name>?param1=value1&param2=value2 and will be processed by the cheat sheet framework.

The following predefined actions are included in the cheat sheet framework:

  • showView - activates view with given view id
  • execute - executes the specified command

Examples:

  • Execute command: <a href="http://org.eclipse.ui.cheatsheet/execute?command=org.eclipse.search.ui.openSearchDialog">Open Search Dialog</a>
  • Show view: <a href="http://org.eclipse.ui.cheatsheet/showView?id=org.eclipse.pde.runtime.LogView">Show Error Log</a>
  • External: <a href="https://eclipse.org">eclipse.org</a>
