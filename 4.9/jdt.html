Debug
Timeout for result of step operation Observing the result of step operations may slow down execution, which may be unusable if the step takes already long. Therefore a timeout (default: 7000 ms) has been introduced after which the observation mechanism is disabled until the end of the step operation.

The timeout can be configured in Preferences > Java > Debug > Don't show if step operation takes longer than (ms).

Option to hide running threads in Debug view New Debug view option has been introduced to show or hide running threads. Hiding running threads can be useful when debugging heavily multithreaded application, when it is difficult to find threads stopped at breakpoints among hundreds or thousands of running threads.
