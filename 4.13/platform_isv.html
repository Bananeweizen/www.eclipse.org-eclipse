Platform Changes
Multiline Support for StringFieldEditor The implementation of StringFieldEditor now provides the option to use multi-line Text widget, suitable for editing larger String values in a preferences page.
Bundle org.eclipse.core.runtime no longer reexports javax.inject The org.eclipse.core.runtime bundle had javax.inject as optional dependency, which it previously reexported, but not anymore. Technically it's not a breaking change as javax.inject was never guaranteed to be there thus whether it is reexported or not was not certain. Eclipse Platform can not guarantee the API stability of this package so it makes no sense to make such a promise. Clients have to Import-Package: javax.inject explicitly if they need it.

SWT Changes
New native implementation of ImageLoader on GTK The implementation of ImageLoader on GTK now uses the native GdkPixbuf API for image loading and saving.

Key highlights of the new ImageLoader on GTK:

  • Loading and saving images has native performance, with a significant speedup especially in loading JPEG files
  • Better GIF animation support
  • Support for more image formats
Reduced ToolItem padding on GTK A fix in ToolItem has greatly reduced the size of some ToolBars on GTK.

Previously, all ToolItems were the same size as the largest ToolItem in a given ToolBar. With this fix, ToolItems are now independently sized, which reduces the total size of ToolBars that contained large items.

EGit's Rebase Interactive view is a good example of this fix. In the following screenshot, the ToolBar shown on top has the fix applied, compared to the ToolBar on the bottom which is much larger.

Search text widgets on Windows To match Cocoa and GTK, the Windows port of SWT gained support for displaying SWT.ICON_SEARCH and SWT.ICON_CANCEL text widget styles.

Input method switching on GTK Early versions of GTK3 allowed the input method to be changed at the application level. This functionality has been deprecated for quite some time, and has now been removed from SWT.

To change the input method, please use the global input method settings available in the desktop environment.
